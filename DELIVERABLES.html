Fase 2 — Architecture Refactor

The project was migrated from Vertical Slice Architecture to Clean Architecture + DDD Domain. Layers: Domain (innermost) → Application (use cases, abstractions) → Infrastructure (EF Core, SMTP, JWT) → API (controllers, Swagger).

See ADR-005 for the decision record.

Fase 2 — Email Notifications

Every service order status transition triggers an automatic email to the customer. The IEmailNotifier abstraction (Application layer) is implemented by SmtpEmailNotifier (Infrastructure layer) and injected into all 7 status-change handlers.

TransitionHandler
RECEIVED → IN_DIAGNOSISStartDiagnosisHandler
IN_DIAGNOSIS → AWAITING_APPROVALSendBudgetHandler
AWAITING_APPROVAL → IN_EXECUTIONApproveServiceOrderHandler
AWAITING_APPROVAL → CANCELLEDRejectServiceOrderHandler
IN_EXECUTION → COMPLETEDCompleteServiceOrderHandler
COMPLETED → DELIVEREDDeliverServiceOrderHandler

Fase 2 — Terraform IaC

Infrastructure defined in infra/ using terraform-aws-modules/eks/aws and terraform-aws-modules/vpc/aws. Provisions VPC, subnets, and EKS cluster on AWS.

Fase 2 — Tests

Unit tests: Domain, Application (all use cases), Infrastructure (SmtpEmailNotifier, email extensions). Integration tests: Auth, Customers, Inventory, Service Order full lifecycle, Health endpoint. CI enforces 80% line coverage on every pull request.
